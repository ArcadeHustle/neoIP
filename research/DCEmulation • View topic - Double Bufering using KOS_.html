It's probably running too fast.
You are directly manipulating the VRAM without any consideration for the framerate of the display. What's probably happening is that the VRAM is changing faster than the raster can update the image, so the raster starts in on another color before getting all the way down the display.

There is a function to wait for the vertical blank period to begin.

From kos-1.2.0/kernel/arch/dreamcast/include/dc/video.h:

Code:
void vid_waitvbl();


However, that's not actually double buffering. KOS does have support for double buffering, but none of the default video modes have multiple pages set up. The only way I've found to make it work involves screwing around with the internals of the video system, which I really don't like doing.

There's two ways to do this. Either manipulate the hardware yourself (which I've done, and it was a mess), or try to make the video driver do it for you (which I've not tried, but should be a lot cleaner). The problem is that KOS provides no way to get a pointer the the back buffer, only the front buffer, which means you need to keep track of it yourself.

Anyway, here's something that should work. It's not tested, so it might be a load of garbage...

Code:
uint16 *backbuffer_s;
uint32 *backbuffer_l;

// Flip page
void dbuffer_flip()
{
    // Copy the active page pointers to the backbuffer pointers
    backbuffer_s = vram_s;
    backbuffer_l = vram_l;

    // Wait for a vertical blank (prevent tearing)
    vid_waitvbl();

    // Tell KOS to flip to the next page
    vid_flip(-1);
}

// Initialise double buffering
void dbuffer_init()
{
    int fb_size;

    // Set the number of video pages to 2
    vid_mode->fb_count = 2;

    // Calculate the size of a single page
    fb_size = vid_mode->width * vid_mode->height;
    switch(vid_mode->pm)
    {
        case PM_RGB555:
        case PM_RGB565:
            fb_size *= 2;
            break;
        case PM_RGB888:
            fb_size *= 4;
            break;
    }

    // Set the start positions of pages 0 and 1
    vid_mode->fb_base[0] = 0;
    vid_mode->fb_base[1] = fb_size;

    // Empty video memory (sets everything to black)
    vid_empty();

    // Perform a page flip
    dbuffer_flip();
}


That should work, I hope. To use it, call dbuffer_init() to set up double buffering. Then, do all your drawing to the backbuffer using the backbuffer_s and backbuffer_l pointers. When you're done, call dbuffer_flip() to flip the pages around. It'll wait until the next vertical blank period, then flip the pages.

Highly inefficient, but it should work.

If not, you can simulate double buffering, by drawing to an in-memory buffer, and copying the contents of that buffer over to video memory when you're done. In some cases, this can actually be faster, because main memory access is a lot faster than video memory access. Basically, you just allocate a chunk of RAM the same size as the framebuffer, then use store queues (or DMA, but that's a bit more difficult) to copy it into video RAM. This should do that:

Code:
uint16 *backbuffer_s = NULL;
uint32 *backbuffer_l = NULL;
static int fb_size = 0;

void dbuffer_flip()
{
    // Wait for a vertical blank (prevent tearing)
    vid_waitvbl();

    // Copy the buffer over using store queues
    sq_cpy(vram_s, backbuffer_s, fb_size);
}

void dbuffer_init()
{
    int nfb_size;

    // Calculate the size of a single page
    nfb_size = vid_mode->width * vid_mode->height;
    switch(vid_mode->pm)
    {
        case PM_RGB555:
        case PM_RGB565:
            nfb_size *= 2;
            break;
        case PM_RGB888:
            nfb_size *= 4;
            break;
    }

    // If the video mode's changed, reallocate it
    if(nfb_size != fb_size)
    {
        free(backbuffer_s);
        backbuffer_s = backbuffer_l = NULL;
    }
    fb_size = nfb_size;

    // Allocate and clear memory, if need be
    if(!backbuffer_s)
    {
        backbuffer_s = (uint16 *)malloc(fb_size);
        backbuffer_l = (uint32 *)backbuffer_s;
        sq_set16(backbuffer_s, 0, fb_size);
    }
}


Again, that should work. Use it the same was as before.
